The Three-Stage Processing Theory

This week was a deep dive into the Information Processing Theory. I learned that humans don't just "see" a button; we process it in stages. According to the lecture (Slide 29), it looks like this:

  1. Sensory Memory: Our eyes grab a snapshot of the UI (very brief!).
  2. Short-Term Memory (Working Memory): We hold the info while we decide what to do (limited capacity).
  3. Long-Term Memory: Where we store things like "Red usually means delete" or "The 'X' is always in the corner."

Mental Models: Why Users Get Confused

A huge takeaway for me was the concept of Mental Models. A mental model is the user's internal map of how they *think* a system works.

If the designer's "System Image" doesn't match the user's "Mental Model," the user gets frustrated. For example, if a user thinks a "Save" button works automatically but the system requires manual clicking, that's a model mismatch.

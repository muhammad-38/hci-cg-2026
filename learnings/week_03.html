The 7-Stage Cycle of Action

According to Donald Norman (1988), we don't just "use" a computer; we go through a cycle to reach a goal. I learned that every interaction can be broken down into these steps:

  1. Goal: What I want (e.g., "I want to turn on Wi-Fi").
  2. Plan: Deciding how to do it.
  3. Specify: Which buttons do I need to click?
  4. Execute: Actually clicking them.
  5. Perceive: Seeing what happened on the screen.
  6. Interpret: Understanding the change (e.g., "The icon turned blue").
  7. Evaluate: Did I get what I wanted?

The "Gulfs" that Break UX

My favorite part of this week was learning about the two "Gulfs." If a user gets stuck, it's usually because of one of these:

  • Gulf of Execution: The user knows what they want, but the UI makes it impossible to figure out how to do it.
  • Gulf of Evaluation: The user did something, but the system didn't give clear feedback, so they don't know if it worked.
